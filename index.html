01 — Datasets

Two Populations, One Problem

The project uses two real-world datasets chosen specifically because they share a clinical task — predicting cardiac events — but differ in almost every other dimension.

Source · Train & Validate

UCI Heart Disease
Multi-Center Dataset

Selected as the training source because it provides a rich set of clinical examination features — many highly discriminative — making it ideal for building strong in-domain classifiers to later stress-test under shift.

Selected as the OOD target because it represents a fundamentally different study design — longitudinal population cohort vs clinical referral — creating a realistic and challenging deployment scenario with structural domain shift.

02 — ML Pipeline

Ten-Phase Experimental Design

Each phase builds on the last — preprocessing feeds baseline training, which sets the reference for every mitigation strategy that follows.

01
Data Preparation
Missing values imputed, duplicates removed, feature spaces aligned, two-layer feature strategy defined.
Done independently on each dataset to prevent information leakage from target to source preprocessing.
→
02
Exploratory Data Analysis
Class distributions, covariate shift (KS tests), feature-by-class breakdowns, correlation structure compared.
Needed to quantify and characterize the domain gap before any modeling — establishes what we're fighting against.
→
03
Baseline In-Domain Training
7 classifiers trained on full UCI feature set with default hyperparameters and 0.50 threshold.
Establishes the in-domain performance ceiling — sets the reference point everything else is measured against.
04
Baseline OOD Evaluation
All 7 models retrained on 4 shared features, evaluated on Framingham.
Directly measures the raw cost of domain shift before any mitigation is applied.
→
05
In-Domain Mitigation
GridSearch tuning, class weighting (balanced), and SMOTE — three source-side strategies tested.
Tests whether source-domain improvements transfer to OOD — spoiler: they largely don't.
→
06
Threshold Tuning
Optimal threshold 0.35 derived from UCI validation, applied to Framingham without using any target labels.
Addresses miscalibration caused by the 3.6× prior shift — the simplest intervention with the biggest recall gain.
→
07
Pooled Training
70% of Framingham combined with UCI for training; 30% held out for unbiased evaluation.
Tests the upper bound of what direct target-domain exposure can achieve under this data regime.
→
08–10
Comparison, Limitations & Conclusion
All strategies consolidated into one table. Limitations documented. Practical deployment implications drawn.
No single strategy dominates all metrics — the right choice depends on clinical objective.

The Domain Gap in Numbers

EDA confirms the shift is multi-dimensional — not just a label rate difference, but distributional divergence across every shared feature, confirmed by statistical tests.

Age Distribution — UCI vs Framingham
UCI patients skew older (mean 53.5) vs Framingham (49.6). KS test: p < 1e-29 — statistically significant covariate shift, meaning age-based decision boundaries learned from UCI will systematically misalign on Framingham.
Shared Feature Means — UCI vs Framingham
Central tendencies appear similar for blood pressure but diverge for cholesterol and age. KS tests confirm significant distributional shape differences beyond what means alone reveal — the overlap hides tail behavior differences that models exploit.

05 — Conclusion

What We Learned

Domain shift between clinical populations is severe, multi-dimensional, and cannot be resolved by source-domain optimization alone.

Finding 01
In-domain AUC is not a deployment indicator
Models achieving AUC 0.90+ on UCI degraded to 0.66–0.69 on Framingham — a gap invisible if only source-domain metrics are reported.
Finding 02
Standard tuning doesn't transfer
GridSearch and SMOTE improved UCI scores but produced negligible OOD gains — fitting the source distribution more precisely does not improve transferability.
Finding 03
Threshold tuning is the strongest source-only fix
Lowering the threshold from 0.50 → 0.35 (UCI validation only, no Framingham labels) raised recall to 0.744 — the best intervention when missing positive cases is costly.
Finding 04
Pooled training shifts the tradeoff, not the ceiling
Adding Framingham training data raised precision to 0.41 but collapsed recall — the dominant negative class suppresses positive predictions, creating a different operating point rather than a better model.
Finding 05
AUC plateau reveals calibration as the core problem
All strategies plateau at 0.66–0.69 AUC, showing domain shift degrades calibration far more than discrimination rank — the problem is how probabilities are scaled, not how samples are ranked.
Finding 06
No single strategy dominates all metrics
Threshold tuning for recall, pooled training for precision, SVM balanced for F1 — the right choice depends entirely on clinical objective and acceptable risk tradeoffs.
Practical Implication
When deploying cardiac risk classifiers across heterogeneous patient populations, source-domain validation metrics are an unreliable indicator of real-world performance. Evaluation on representative target-domain data — and explicit awareness of label distribution differences — is essential for responsible clinical deployment.
